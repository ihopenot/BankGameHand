# MainLoop

初始化游戏状态

## Warmup 阶段

在玩家接手前自动运行，让经济达到运转中的稳态。

### 初始化

- 生成所有公司（随机CEO特质）
- 生成工厂（每公司一个初始工厂）
- 设定商品初始价格（基于生产成本的合理标价）
- 设定经济周期初始相位 φ（随机）
- 居民初始储蓄

### 自动运行 N 回合（如 20~50 回合）

- 执行主循环（采购→生产→决策→结算）
- **没有银行，不允许贷款**
- 公司纯靠自有资金运转
- 破产正常触发，弱公司自然淘汰
- 经济周期正常波动

### Warmup 结束

- 玩家银行加入，获得初始资金
- 继承当前经济状态：公司、库存、价格、存款、经济指数全部保留
- 公司从此可以申请贷款

### 每局随机性来源

| 来源 | 效果 |
|------|------|
| 经济周期相位 φ | 玩家接手时可能处于繁荣、衰退或任意阶段 |
| CEO特质随机 | 每局公司性格组合不同 |
| Warmup中的淘汰 | 不同公司存活，市场格局不同 |
| 噪声积累 | 供需、价格、库存状态每局不同 |

## 主循环

- 回合开始
- 更新经济指数
- 采购阶段
    - 居民部门采购
    - 公司采购
- 公司生产
- 公司决策
    - 产品定价
    - 工资发放决策
    - 品牌/科技投入决策
    - 计算本回合总开支需求
    - 提交贷款申请（扩产/流动资金/紧急，基于预计现金缺口）
- 玩家操作
    - 查看经济数据
    - 设定存款利率
    - 审批所有贷款申请
    - 银行间拆借
- 贷款发放
- 公司投资（建厂/扩产）
- 回合结算
    - 收入到账
    - 支付所有开支（采购赊账、工资、利息、仓储、维护、品牌/科技支出）
    - 贷款还款
    - 检查现金流：现金 < 0 → 清算
    - 检查净资产：净资产 < 0 → 强制清算
    - 市场补充：检查是否需要政府扶持新公司进入
    - 游戏结束判定
- 回合结束

## 市场补充机制

防止产业链断裂导致经济不可恢复的死局。

```
每回合结算后检查：
    对每种商品：
        如果 该商品存活生产者数量 < 最低阈值（如 2）:
            政府出资成立新公司：
            - 获得一个基础工厂（最低等级）
            - 获得启动资金（足够运转数回合）
            - CEO特质随机生成
            - 无品牌、无科技积累、无客户基础
```

**设计理由：** 模拟现实中市场空白吸引新进入者。新公司从零开始，不会破坏现有竞争平衡，但保证每条产业链至少有基本的供给能力，避免下游因上游全灭而连锁崩溃。

## 游戏结束条件

```
每回合结算后检查：

强制结束：
    - 存活银行数量 < 2 → 游戏结束
    - 所有银行净资产为负 → 游戏结束

正常结束（开局可选模式）：
    - 模式A：固定回合 → 达到 N 回合时结束，净资产最高者获胜
    - 模式B：目标触发 → 某银行先达到目标净资产时结束

胜负判定：
    - 正常结束：净资产最高的玩家获胜
    - 强制结束（仅一家存活）：存活者获胜
    - 强制结束（全部资不抵债）：亏损最少者"惨胜"
    - 银行破产即淘汰，不参与最终排名
```

## 采购阶段

### 机制：挂牌价 + 逐轮匹配

卖方定价挂牌，买方按偏好逐轮选择卖方下单，成交价为卖方标价。每轮满足的买方和售罄的卖方退出，剩余参与者继续下一轮，直到匹配结束。

**设计理由：** 供应链中的原料和中间品是同质或近似同质的商品，现实中走挂牌价/议价模式而非拍卖。密封拍卖适用于独一无二的标的物（艺术品、政府合同等），在 B2B 大宗商品场景下不自然。挂牌价模型更简洁，且公司差异化（品质、品牌、CEO特质）通过卖方定价行为和买方偏好排序充分体现。

### 数据结构

**卖方挂单（SellOrder）：**
- 卖方公司引用
- 商品（种类 + 质量 + 品牌）
- 标价（卖方定价，即成交价）
- 可售数量（本回合库存）

**买方采购意向（BuyIntent）：**
- 买方公司引用
- 目标商品种类（如"硅"）
- 需求量（基于产能和库存缺口）
- 供应商偏好排序（由评分公式计算，每个买方独立）

**供应商评分公式（每个买方独立计算）：**

品质和价格统一为"性价比"维度，与品牌组合：

```
性价比 = 品质产出系数 / 标价

供应商评分 = w_性价比 * 性价比 + w_品牌 * 品牌值

w_品牌 ∝ CEO营销意识
w_性价比 = 1 - w_品牌
```

不同买方因 CEO 特质不同，对同一组卖方的评分排序可能不同：
- 高营销意识 CEO → 品牌权重高 → 偏好品牌供应商
- 低营销意识 CEO → 几乎纯看性价比 → 偏好便宜供应商

### 流程

**步骤1：卖方挂单**

每个拥有库存的卖方公司生成 SellOrder，公布商品信息 + 标价 + 可售数量。

**步骤2：买方生成偏好排序**

每个买方根据自己的评分公式，对所有同类卖方排序，生成供应商偏好列表。

**步骤3：逐轮匹配**

```
初始化：
    剩余买方 = 所有有需求的买方
    剩余卖方 = 所有有库存的卖方

每轮：
    1. 每个剩余买方向自己偏好列表中排名最高的剩余卖方发送订单
       订单量 = 该买方的剩余需求量

    2. 每个卖方收集本轮所有订单，与自己的剩余库存对比：

       如果 总订单量 ≤ 剩余库存（供大于求）：
           全部成交，成交价 = 卖方标价
           被满足的买方退出
           卖方剩余库存扣减，留在场内

       如果 总订单量 > 剩余库存（供小于求）：
           按各订单量等比例分配库存
           成交价 = 卖方标价
           卖方售罄退出
           买方部分满足，剩余需求留在场内继续

       现金不足部分记为应付账款（见 Company.md 赊账机制）

    3. 终止判定：
       如果 无剩余买方 → 结束
       如果 无剩余卖方 → 结束，剩余需求记为缺货
       如果 本轮无任何成交 → 结束
```

**步骤4：未满足需求处理**

- 买方汇总从所有卖方实际买到的总量
- 总量 < 工厂满产需求 → 生产阶段按比例减产
- 未售出库存留在卖方，产生仓储成本

### 匹配过程示例

```
卖方A(便宜, 库存100)  B(均衡, 库存80)  C(精品, 库存50)
买方甲(需求90, 偏好A>B>C)  乙(需求70, 偏好C>B>A)

第一轮：
  甲→A(订单90)，乙→C(订单70)
  A: 90 ≤ 100 → 甲全部满足退出，A剩余10
  C: 70 > 50  → 等比分配，乙买到50，C售罄退出，乙剩余需求20

第二轮：
  乙(剩余20)→B（乙的次选）
  B: 20 ≤ 80 → 乙全部满足退出，B剩余60

结束。A剩余10库存，B剩余60库存（产生仓储成本）。
```

### 算法性质

- **收敛性**：每轮至少有买方退出或卖方退出，最多 min(B, S) 轮
- **复杂度**：O(R × (B + S))，其中 R = 轮数 ≤ min(B, S)，游戏规模下完全无性能问题
- **确定性**：等比例分配无需 tie-breaking 规则，结果完全确定
- **公平性**：偏好不同的买方互不干扰，偏好相同时按需求量公平分配

### 与生产阶段衔接

- 原料来自多个卖方时，按加权平均计算等效质量：
```
等效质量 = sum(各批次质量 * 各批次数量) / 总数量
```
- 等效质量影响下游产出（如高品质硅增加处理器产出比例）
- 原料不足时按比例减产（固定成本照付，单位成本上升）

### 与公司决策阶段衔接（卖方定价）

卖方观察本回合销售情况调整下回合标价（详见 [Company](Company.md) 决策一）：
- 库存售罄 → 涨价（幅度受 CEO 利润追求影响）
- 库存剩余较多 → 降价（幅度受 CEO 风险偏好影响）
- 调价噪声受 CEO 洞察力影响（低洞察力 → 定价偏离市场）

涌现行为：
- **价格战**：同行业公司互相降价抢客户 → 利润率下滑 → 贷款偿还压力增大
- **品牌溢价**：高品牌公司定价更高但仍有忠实客户 → 利润率更健康
- **定价失误的惩罚**：低洞察力 CEO 定价过高 → 卖不出去 → 库存积压 → 现金流恶化
- **供给冲击传导**：上游公司破产 → 下游买不到原料 → 产量下降 → 终端消费品短缺 → 价格上涨
- **客户分流**：不同偏好的买方自然流向不同类型的卖方，品牌厂和低价厂各有客群

### 库存成本

- 未售出库存每回合产生仓储成本
- 计入现金流量表支出
- 通过 Transaction 接口扣除

### 所有交易通过 Transaction 接口

- 采购成交：买方付现金（卖方标价），卖方交商品
- 仓储费用：公司付现金（运营支出）
