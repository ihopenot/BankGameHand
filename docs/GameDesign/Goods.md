# Goods

## 商品分类

每个商品有一个产业链位置标签：

| 位置 | 说明 | 买方 |
|------|------|------|
| 原料 | 产业链顶端，无需原料生产（采掘/种植） | 中游公司 |
| 中间品 | 由原料加工而成 | 下游公司 |
| 终端消费品 | 由中间品加工而成 | 居民 |

## 商品属性

| 属性 | 说明 | 决定方式 |
|------|------|---------|
| 种类 | 如硅、芯片、手机 | 固定 |
| 质量 | 影响下游良品率（最高+10%产量） | 生产公司的科技值 |
| 品牌 | 影响采购/消费竞争力评分 | 公司品牌投入，每层独立 |
| 价格 | 卖方定价 | 公司决策AI |
| 品质上限 | 该商品种类可达到的最高品质 | 固定配置 |

## 产业链

三层结构，多链并行，不交叉。

```
产业链A（电子）:  硅矿(原料) → 芯片(中间品) → 手机(终端消费品)
产业链B（纺织）:  棉花(原料) → 布料(中间品) → 服装(终端消费品)
产业链C（食品）:  小麦(原料) → 面粉(中间品) → 食品(终端消费品)
```

每条链对应不同的居民需求特征：

| 终端消费品 | 经济敏感度 | 对应居民层级需求 |
|-----------|-----------|----------------|
| 食品 | 低（刚需） | 三层都有需求 |
| 服装 | 中（弹性） | 中收入+高收入为主 |
| 手机 | 高（弹性） | 中收入+高收入为主 |

## 原料层特殊处理

原料公司不需要采购输入，直接生产：

```
实际产出量 = 基础生产量（不受原料充足率影响）
产出品质 = 科技排名系数 * 该原料品质上限
```

生产成本只有维护成本和工资，没有原料采购成本。

## 品牌机制

品牌值每层独立计算，不跨层传导。

```
品牌值(t) = 品牌值(t-1) * 衰减率 + 本回合品牌投入带来的增量

衰减率 < 1（品牌不维护会自然衰减）
品牌增量 = f(品牌支出)（递减增长，投入越多边际效果越小）
```

品牌不投入就会衰减，持续投入才能维持。

## 商品种类配置结构

每种商品在游戏初始化时配置：

```
{
    名称: "芯片",
    产业链位置: "中间品",
    品质上限: 100,
    所属产业链: "电子",
    对应工厂类型: "芯片工厂",
    输入原料: "硅",
    满产需求量: 200,
    基础生产量: 100
}
```
