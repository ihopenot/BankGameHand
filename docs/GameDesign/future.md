# Future - 暂缓实现的设计

本文档记录设计上已有思路但暂不纳入当前版本的机制。

## 挤兑机制

银行准备金率过低时触发居民恐慌性取款，形成死亡螺旋。

### 设计概要

在现有的两层取款机制（消费取款 + 信心取款）之上增加第三层：

**第三层：挤兑（银行出问题时）**

当某银行准备金率过低，居民恐慌性取款：
```
如果 某银行准备金率 < 挤兑警戒线:
    挤兑取款 = 该银行存款 * 挤兑比例
    挤兑比例随准备金率下降而急剧上升
```

挤兑会形成死亡螺旋：准备金不足 → 挤兑 → 准备金更不足 → 更多挤兑。

取款总额变为：
```
本回合取款 = 消费取款 + 信心取款 + 挤兑取款

如果 取款总额 > 银行现金（准备金）:
    实际取款 = 银行现金（银行被掏空）
    准备金率归零 → 下回合挤兑加剧
```

### 实现时需要解决的问题

1. **终止条件**：挤兑在同一回合内不应迭代计算，否则无限循环。建议每回合基于回合开始时的准备金率快照计算一次，死亡螺旋跨回合展开。
2. **挤兑比例函数**：需要具体曲线定义（如二次函数），需要确定警戒线和最大挤兑比例参数。
3. **挤兑传染效应**：一家银行挤兑是否影响其他银行的存款人信心（经济学家建议引入）。
4. **玩家应对手段**：挤兑发生时玩家可以通过银行间拆借紧急补充准备金。

### 玩法价值

- 创造"银行也可能完蛋"的紧张感，打破旁观者视角
- 准备金率管理成为核心策略维度
- 与经济周期联动：衰退期信心取款增加 → 准备金承压 → 可能触发挤兑
- 涌现行为：死亡螺旋、银行间传染、挤兑引发的系统性危机

## 随机事件系统

每若干回合触发随机事件，对特定产业链或全局经济产生冲击，打破经济周期的可预测性，增加每局的叙事感和重玩价值。

### 设计概要

```
每回合有一定概率（如 5~10%）触发一个随机事件
事件从预设事件池中抽取
每个事件有：影响范围、持续回合数、效果强度
```

**事件类型示例：**

| 事件 | 影响范围 | 效果 |
|------|---------|------|
| 技术突破 | 某产业链 | 该链产品品质上限提升，刺激需求 |
| 原料短缺 | 某原料 | 该原料产量下降，上游价格上涨 |
| 消费潮流 | 某终端商品 | 需求激增若干回合 |
| 食品安全事件 | 食品链 | 食品需求短期下降，品牌值高的公司受影响小 |
| 金融丑闻 | 某银行 | 该银行存款流失加速 |
| 基建投资 | 全局 | 政府临时增加保底收入或补贴 |

### 与经济周期的关系

事件叠加在经济周期之上，可以放大或抵消周期效应：
- 繁荣期 + 消费潮流 → 经济过热加剧
- 衰退期 + 原料短缺 → 雪上加霜
- 衰退期 + 技术突破 → 提前复苏

### 玩法价值

- 打破纯数学推算经济周期的支配策略
- 每局不同的事件序列创造独特的经济叙事
- 玩家需要对突发情况做出快速反应（调整放贷策略、收缩/扩张）
- 同类游戏（维多利亚、外星贸易公司、纪元）的标配机制

## 扩展银行操作

当前玩家每回合操作为：存款利率、贷款审批、国债买卖、银行间拆借。以下是可在后续版本增加的操作维度：

### 尽职调查（信息投资）

花费资金调查特定公司，解锁隐藏信息（CEO特质、真实供需感知偏差、详细现金流预测）。当前所有银行看到相同的公开财报，引入尽职调查后产生信息不对称——更了解某家公司的银行可以更准确地定价风险。

### 贷款管理（存量操作）

对已发放的贷款做主动管理：
- 提前催收：要求借款人提前还款（可能损害客户关系）
- 展期续贷：延长还款期限（降低当期还款压力，但风险敞口延长）
- 贷款转让：将贷款打折卖给其他银行（快速回收流动性，但损失利润）

### 银行声誉/营销

花钱提升银行声誉，影响存款吸引力。存款分配不仅靠利率，还靠声誉——为"低利率但高声誉"的策略路线提供可能。

## 金融加速器（抵押品机制）

引入贷款抵押品，使资产价格与信贷互相强化，产生正反馈循环（Bernanke & Gertler, 1989）。

**核心机制：** 公司以工厂或库存作为抵押物申请贷款，贷款额度受抵押物估值约束。经济繁荣时资产升值 → 抵押品价值上升 → 可贷额度增加 → 更多投资 → 资产进一步升值；衰退时反向螺旋。

**对玩法的影响：**
- 银行审批贷款时多了一个维度：不仅看公司现金流，还要评估抵押品价值是否充足
- 经济下行时抵押品贬值可能触发"追加保证金"，加剧企业困境
- 集中破产时抛售资产（fire sale）压低资产价格，进一步恶化其他公司的抵押品价值

**实现复杂度高：** 需要资产估值模型、抵押率设定、追保机制、抛售折价动态化等完整子系统。

## 跨产业链关联

当前三条产业链（电子/纺织/食品）完全独立，一条链出问题不影响其他链。引入共享投入品（如"能源"或"物流"），使产业链间产生关联，增强经济波动的跨产业传导。

**可能的实现方式：**
- 新增一种所有工厂都需要的共享投入品（如能源），该投入品价格波动同时影响三条链的生产成本
- 或将工厂输入从单一原料扩展为原料列表，允许部分原料跨链共享

**玩法价值：** 系统性风险深度增加——一条链的供给冲击可以传导到其他链，银行需要关注跨产业风险敞口而非只看单条链的健康度。

## 其他待考虑机制

### 银行差异化/专业化路径

当前所有银行功能完全相同。可引入银行特色或升级树——如"产业银行"（贷款给特定产业链有利率优惠）、"零售银行"（吸收存款能力更强）、"投行"（可参与公司股权投资）。增加策略多样性和重玩价值。

### 动态政府政策

当前政府政策（价格管制、产业补贴）局内固定不变。可让政府在极端经济状况下动态调整：衰退期增加补贴、通胀期收紧管制。模拟自动稳定器效果，增加不确定性。

### CEO 学习/适应机制

当前 CEO 特质固定不变，公司不会从失败中学习。可引入缓慢的特质漂移——长期存活的公司洞察力逐渐提升，连续亏损的公司风险偏好逐渐下降。使长期游戏中公司行为更丰富。

### 价格粘性/调价成本

当前公司每回合可自由调价，价格完全弹性。可引入调价成本或调价概率限制（新凯恩斯主义菜单成本理论），创造短期价格粘性，使利率变化对实体经济的影响更显著。

### 贷款展期

当前贷款还款是刚性的，到期必须还。可允许银行选择对困难但有前景的公司展期续贷，减少"技术性破产"（公司长期可行但短期现金流断裂）。增加银行的存量贷款管理深度。

### 行为经济学建模

在公司 AI 决策中引入更多行为经济学特征：
- **损失厌恶**：亏损时行为与盈利时不对称（如更激进降价清库存）
- **锚定效应**：定价过度依赖上一回合价格，市场剧变时调整不够
- **沉没成本谬误**：对已投入大量资金的项目不愿放弃

### 网络多人架构

多人游戏的基础设施，设计文档完全未涉及。需要确定：C/S 还是 P2P、状态同步机制、断线重连、玩家超时处理（AFK 自动跳过或 AI 代操）。

### 回放与存档系统

回合制架构天然适合回放——记录每回合所有玩家操作 + 随机种子即可完美重现。同时服务于调试、测试和玩家回看。GameContext 的序列化方案需要定义。
