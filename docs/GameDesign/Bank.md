# Bank

银行，玩家角色。持有 [EconomyEntryComponent](EconomyEntryComponent.md)，是标准经济实体。

## 银行资产负债表

| 资产 | 负债 |
|------|------|
| 现金（准备金） | 居民存款 |
| 贷款（应收） | 银行间借款 |
| 政府债券 | 所有者权益（净资产） |

## 业务一：企业贷款（核心玩法）

### 贷款申请类型

| 类型 | 触发时机 | 特征 |
|------|---------|------|
| 扩产贷款 | 公司决策阶段决定建厂/扩产 | 金额大、期限长、金额不足则放弃扩产 |
| 流动资金贷款 | 公司预计现金流紧张 | 金额中等、期限短、金额不足可接受部分 |
| 紧急贷款 | 公司预计结算后现金为负 | 金额=预计现金缺口、未获批则结算时清算 |

### 流程

```
1. 公司决策阶段：计算总开支需求，提交贷款申请
2. 玩家操作阶段：各银行查看申请并报价
3. 贷款发放：公司选最优报价组合，资金到账
```

### 贷款申请信息（各银行都能看到）

```
- 申请公司
- 申请金额
- 申请类型（扩产/流动资金/紧急）
- 贷款期限（回合数）
- 公司公开财报：
    - 资产负债表
    - 近几回合利润
    - 产能利用率
    - 库存情况
    - 现有贷款总额
```

### 银行报价（玩家操作）

每个银行对每个申请独立报价：

```
- 是否批准
- 批准金额（可低于申请金额）
- 贷款利率（>= 标准利率）
- 贷款期限（可调整）
```

### 公司选择（联合放贷）

公司从所有批准的报价中，按利率从低到高依次填充需求金额：

```
将所有批准报价按利率从低到高排序
依次累加金额，直到满足贷款需求或报价用完

例：申请1000万
  银行A：600万，利率7% → 先用A的600万
  银行B：500万，利率8% → 再用B的400万（部分使用）
  总计1000万，加权利率 = 7.4%
```

### 金额不足时的处理

| 类型 | 金额不足时 | 理由 |
|------|-----------|------|
| 扩产贷款 | 放弃，不建厂 | 建一半工厂没意义 |
| 流动资金贷款 | 接受部分金额 | 有多少用多少 |
| 紧急贷款 | 接受部分金额 | 能补多少补多少，缺口仍填不上则结算时清算 |

### 利率上限（防恶意报价）

公司有利率接受上限，超过自动拒绝：

```
利率接受上限 = 标准利率 * 倍数上限（如 3 倍）

实际上限 = 基础上限 * (1 + CEO风险偏好 * 系数)
```

- 激进CEO：急需资金扩张，能接受更高利率
- 保守CEO：利率稍高就宁可不借
- 紧急贷款时利率上限更宽松（公司更绝望）

### 还款机制

```
每回合还款额 = 本金 / 贷款期限 + 剩余本金 * 贷款利率

例：贷款1000，期限5回合，利率8%
  第1回合：200 + 1000*8% = 280
  第2回合：200 + 800*8% = 264
  第3回合：200 + 600*8% = 248
  ...
```

在回合结算阶段统一还款。

### 违约处理

```
如果公司回合结算后现金 < 0：
    进入清算流程，银行贷款按优先级3偿还
    无法收回部分 → 坏账核销
```

## 业务二：居民存款

三个居民阶层的储蓄可以存入银行。多家银行时居民按存款利率吸引力分配：

```
吸引力模型：
以最高利率银行吸引力 = 1
利率每低于最高利率1%，吸引力减少 decay_per_percent（可配置，默认0.2）
吸引力 = max(0, 1.0 - (最高利率 - 该银行利率) * decay_per_percent)

某银行吸收的存款 = 总居民储蓄 * (该银行吸引力 / 所有银行吸引力之和)
所有银行吸引力为0时等额分配
```

**玩家决策：** 设定存款利率
- 利率高 → 吸引更多存款 → 可用资金多 → 但利息支出大
- 利率低 → 存款少 → 可放贷金额受限 → 但成本低

**存款利息：**
```
每回合银行支付：存款总额 * 存款利率
```

### 居民取款机制

两层取款触发，逐层叠加：

**第一层：消费取款（常规）**

每回合都会发生，可预测：
```
如果 当期收入 < 消费预算:
    消费取款 = 消费预算 - 当期收入
```

**第二层：信心取款（恐慌囤现金）**

经济衰退时居民不信任银行，想把钱拿在手里：
```
信心取款比例 = max(0, -消费信心指数) * 信心取款系数

信心取款 = 存款余额 * 信心取款比例
```

信心为正时不触发，信心越负（衰退越深）取款越多。

**取款总额：**
```
本回合取款 = 消费取款 + 信心取款

如果 取款总额 > 银行现金（准备金）:
    实际取款 = 银行现金（银行被掏空）
```

> **注：** 挤兑机制（准备金率过低时的恐慌性取款和死亡螺旋）暂不实现，留待后续版本引入。详见 [future](future.md)。

### 玩家应对策略

| 取款层级 | 银行的应对 |
|---------|-----------|
| 消费取款 | 可预判，保持合理准备金即可 |
| 信心取款 | 关注经济周期，衰退前提前收缩放贷、增加准备金 |

## 业务三：政府债券

银行可用闲置资金购买政府债券，利率低但零风险。

```
债券利率 = 标准利率 * 债券利率系数（如 0.5~0.7）
```

债券利率始终低于标准利率，确保放贷的收益率更高。但债券零风险——不存在违约和坏账。

**玩家操作：** 每回合可买入/卖出政府债券
- 买入：消耗现金，增加债券持有量
- 卖出：债券按面值赎回，回收现金（无折价，政府信用兜底）

**利息收入：**
```
每回合债券利息 = 持有债券总额 * 债券利率
```

**策略权衡：**
- 资金放贷 → 高收益但有坏账风险，且资金被锁定在贷款期限内
- 资金买国债 → 低收益但零风险，可随时卖出回收流动性
- 经济繁荣期：放贷收益远高于国债，国债吸引力低
- 经济衰退期：坏账风险高，国债成为避风港
- 准备金不足时：卖出国债即时回收现金，比拆借更快更便宜

## 业务四：银行间拆借

银行之间可以短期借贷资金，期限固定一回合，下回合自动还本付息。

### 流程

```
玩家操作阶段：

1. 借方发起请求：
   - 指定借入对象（哪家银行）
   - 借入金额
   - 提出利率

2. 被请求的银行收到请求后决策：
   - 接受：资金立即划转
   - 拒绝：无事发生
   - 可同时收到多家银行的借入请求，逐一决策

3. 一个玩家可以同时：
   - 向其他银行发起借入请求
   - 对收到的借入请求做出接受/拒绝决策
```

### 还款

```
下一回合结算时自动执行：
    借方偿还 = 本金 + 本金 * 拆借利率
    通过 Transaction 接口划转

如果借方现金不足以偿还：
    欠款部分记为逾期
    逾期金额在借方回合结算时作为负债处理
    如果借方因此现金 < 0 → 触发清算，拆借款按银行间债务处理
```

### 利率约束

```
拆借利率范围 = [0, 标准利率 * 上限倍数（如 3 倍）]
```

无下限约束——允许低息甚至零息拆借（可能出于策略性合作目的）。

### 策略维度

| 场景 | 借方视角 | 出借方视角 |
|------|---------|-----------|
| 衰退期准备金紧张 | 紧急借入维持流动性 | 趁人之危收取高息，或拒绝加速对手危机 |
| 繁荣期资金充裕 | 借入扩大放贷规模 | 借出闲置资金赚取利差 |
| 对手即将破产 | — | 拒绝拆借加速其出局，但对方破产则拆借款成坏账 |

涌现行为：
- **危机博弈**：对手急需资金时，是救还是杀？救了对手活下来继续竞争，不救则对手破产但自己的拆借款可能已经借出去了
- **策略性合作**：两家银行低息互拆，互相做流动性后盾，联合对抗第三方
- **试探信息**：向对手发起拆借请求可以观察其反应——秒拒说明对方也缺钱，秒接说明对方资金充裕

## 银行破产

**触发条件：**
```
银行净资产 = 总资产 - 总负债（存款 + 拆借借款）

如果 净资产 < 0 → 银行破产，该玩家出局
```

**破产影响：**
- 该银行的居民存款按清算回收率部分损失（不自动接管给其他银行）
- 该银行的贷款客户（公司）的未偿还贷款自动核销，后续融资需自行向其他银行申请
- 其他银行借给该银行的拆借款成为坏账

## 银行收入与支出

```
收入 = 贷款利息收入 + 债券利息收入 + 拆借利息收入
支出 = 存款利息支出 + 拆借利息支出 + 坏账核销

利润 = 收入 - 支出
```

核心盈利模式是利差：贷款利率 > 存款利率。

## 胜负条件

详见 [MainLoop](MainLoop.md) 游戏结束条件。

银行破产即淘汰，不参与最终排名。

## 玩家每回合操作

```
1. 查看经济指数、公司财报、市场数据
2. 设定存款利率
3. 审批所有贷款申请（批准/拒绝/调整金额和利率）
4. 买入/卖出政府债券
5. 银行间拆借操作（借入/借出）
6. 确认结束回合
```
