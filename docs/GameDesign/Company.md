## CEO特质

全部取值 0~1。

| 特质 | 含义 | 影响的决策 |
|------|------|-----------|
| 商业洞察力 | 对市场供需的判断准确度 | 定价合理度、投资时机判断 |
| 风险偏好 | 激进(高) vs 保守(低) | 投资扩产、借贷意愿 |
| 利润追求 | 高利润率(高) vs 市场份额(低) | 产品定价策略 |
| 营销意识 | 重视品牌(高) vs 忽视品牌(低) | 品牌投入 |
| 科技重视度 | 重视研发(高) vs 忽视研发(低) | 科技投入 |

### CEO 性格组合涌现出的公司类型

| 性格组合 | 公司类型 | 特征 |
|---------|---------|------|
| 高风险 + 高利润 | 狼性公司 | 激进扩张、高价卖，暴利或暴死 |
| 低风险 + 低利润 | 稳健老厂 | 保守经营、薄利多销，活得久但赚得少 |
| 高洞察 + 高营销 | 精品公司 | 精准定价、重品牌，利润率高 |
| 高科技 + 高营销 | 科技品牌 | 高品质+强品牌，产品竞争力极强但投入大 |
| 低洞察 + 高风险 | 赌徒公司 | 看不清市场还爱冒险，容易破产，银行噩梦 |

## 公司决策AI

### 决策一：产品定价

每回合卖方调整下回合标价。

```
目标利润率 = 基础利润率 * (1 + 利润追求 * 利润系数)

如果 本回合库存售罄:
    涨价幅度 = 基础涨幅 * (1 + 风险偏好 * 风险系数)
如果 本回合库存有剩余:
    降价幅度 = 基础降幅 * (1 + (1 - 利润追求) * 让利系数)

新标价 = 旧标价 + 涨价/降价幅度 + 噪声
噪声幅度 = f(1 / 商业洞察力)

价格受政府价格管制钳制在 [下限, 上限] 内
```

性格差异：
- 高利润追求 → 定价偏高，宁可少卖也要高毛利
- 低利润追求 → 定价偏低，薄利多销抢市场份额
- 高洞察力 → 调价幅度精准贴合市场
- 低洞察力 → 可能过度涨价或降价不够

### 决策二：投资（建厂/扩产）

```
投资意愿 = f(风险偏好, 现金储备, 市场前景)

市场前景 = 该公司感知的供需比（受洞察力影响）

如果 投资意愿 > 阈值 且 有足够资金或贷款能力:
    决定建新工厂 / 扩产
```

性格差异：
- 高风险偏好 → 市场稍有增长就扩产，甚至借钱扩产
- 低风险偏好 → 只有在现金充裕且市场明确增长时才扩产
- 高洞察力 → 对市场前景判断准确，扩产时机好
- 低洞察力 → 可能在顶峰扩产（高位接盘）或在低谷畏缩（错失机会）

### 决策三：工资发放

```
工资总额 = 基础工资支出 * 产能利用率
```

产能利用率高 → 工资高，减产 → 工资缩减。纯公式驱动，不受CEO性格影响。

### 决策四：原料采购偏好

每个买方根据 CEO 特质独立计算供应商评分，决定采购优先级（详见 [MainLoop](MainLoop.md) 采购阶段）：

```
性价比 = 品质产出系数 / 标价（越高越好）

供应商评分 = w_性价比 * 性价比 + w_品牌 * 品牌值

w_品牌 ∝ 营销意识
w_性价比 = 1 - w_品牌
```

所有公司都理性追求高性价比，差异仅在品牌权重上：
- 高营销意识 → 更看重品牌供应商，愿意付品牌溢价
- 低营销意识 → 几乎纯看性价比

采购阶段每个买方按自己的评分排序向卖方下单，通过逐轮匹配分配。不同偏好的买方自然流向不同类型的卖方，买方无法直接控制能买到多少（取决于市场供需和竞争），但偏好排序决定了优先从谁那里买。

### 决策五：品牌投入

```
品牌支出 = 营收 * 品牌投入比例
品牌投入比例 = 基础比例 * (1 + 营销意识 * 营销系数)

品牌支出 → 提升商品的品牌值（具体机制留到商品体系设计）
```

### 决策六：科技投入

```
科技支出 = 营收 * 科技投入比例
科技投入比例 = 基础比例 * (1 + 科技重视度 * 科技系数)

科技支出 → 提升公司科技值（累积在无形资产中）
```

**科技值对产出品质的影响（同行归一化）：**

```
科技排名系数 = 该公司科技值 / 同类商品所有公司中最高科技值

产出品质加成 = 科技排名系数 * 10%
```

科技值最高的公司获得满额 10% 产出加成，其他公司按比例。

涌现行为：
- 军备竞赛：你不投研发不会死，但对手投了你就落后
- 内卷陷阱：所有公司都猛投研发时，大家花了很多钱但谁也没优势，利润被研发吃掉
- 一家独大时收益最高，竞争者追上来后优势稀释

## 回合资金流程

```
采购阶段：
    公司按需采购，现金不足部分记为应付账款（赊账）

生产阶段：
    正常生产

公司决策：
    产品定价、工资发放、品牌/科技投入决策
    计算本回合总开支需求
    提交贷款申请（基于预计现金缺口）

玩家操作：
    银行审批贷款

贷款发放：
    资金到账

公司投资：
    建厂/扩产

回合结算：
    1. 收入到账（卖出商品的货款）
    2. 计算所有应付总额（采购赊账、工资、利息、仓储、维护、品牌/科技支出、贷款还款）
    3. 破产检查：
       如果 现金 < 应付总额 → 标记破产，进入清算
       如果 净资产 < 0 → 强制清算
    4. 对未破产公司执行所有支付
    5. 贷款还款
```

## 赊账机制

采购阶段现金不足时，允许赊账采购，回合结算时统一还款。

```
买方：资产负债表 → 负债 → 应付账款（欠卖方的钱）
卖方：资产负债表 → 资产 → 应收账款（买方欠的钱）
```

买方破产时，卖方的应收账款可能无法全额收回。

## 破产机制

### 触发条件一：现金流断裂

回合结算时，支付所有开支和贷款还款后现金为负，进入清算。

```
如果 回合结算后现金 < 0:
    进入清算
```

注：公司在决策阶段已预计现金缺口并申请贷款，玩家已审批。结算时仍然现金不足说明贷款未获批或金额不够，直接清算。

### 触发条件二：资不抵债（强制清算）

```
净资产 = 总资产 - 总负债

如果 净资产 < 0:
    强制进入清算，无法求助银行
```

### 清算流程

回合结算时，所有满足清算条件的公司在同一轮中处理，**不递归触发连锁清算**：

```
1. 所有公司同时执行：收入到账 → 支付开支 → 贷款还款
2. 检查所有公司的现金和净资产
3. 收集所有满足清算条件的公司（本轮破产名单）
4. 按净资产从低到高排序，依次执行清算：
   a. 停止生产，停止采购
   b. 变卖所有资产：
      - 工厂以折价出售（如造价的 50%）
      - 库存商品以当前市场价的折价出售（如 70%）
      - 现金直接计入
   c. 变卖所得按优先级偿还：
      优先级1：员工工资（欠薪）
      优先级2：应付账款（上游供应商的货款）
      优先级3：银行贷款（玩家的钱）
   d. 偿还不完的部分 → 银行坏账 / 卖方坏账
5. 坏账立即记入受影响的公司和银行的资产负债表
6. 本回合不再重新检查其他公司是否因坏账触发清算
   → 受波及的公司如果因此资不抵债，下一回合结算时自然满足清算条件
```

**不递归的理由：**
- 连锁破产的传导被"展开"到多个回合，每回合最多一波清算
- 玩家在两波清算之间有决策窗口（给受波及的公司发紧急贷款）
- 实现简单，无需递归处理，无边界条件风险

### 清算对其他系统的影响

| 影响 | 具体效果 |
|------|---------|
| 供给减少 | 该公司的产品退出市场，同行竞争压力降低 |
| 银行坏账 | 贷款无法全额收回，影响玩家资产 |
| 卖方坏账 | 应收账款无法全额收回 |
| 居民工资 | 公司工资总额下降 → 居民收入减少 |
| 上游需求 | 下游公司破产 → 上游少了一个买方 |

### 破产前信号（玩家可观察）

| 信号 | 含义 |
|------|------|
| 连续多回合库存积压 | 产品卖不动 |
| 现金持续下降 | 入不敷出 |
| 负债率上升 | 借钱越来越多 |
| 产能利用率低 | 买不到料或需求不足 |
